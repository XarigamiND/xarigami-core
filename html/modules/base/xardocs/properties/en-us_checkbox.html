Description

The Checkbox property provides a field that can be checked or unchecked by the user. It can also have a value separate from the checked status.

On output display the Checkbox will display 'Yes' or 'No', or other values set for checked or unchecked values.

Attributes

  • hasvalue: [default='Yes']
    Sets a default value for the checked status used on output display of the checkbox value.
  • hasnovalue: [default='No']
    Sets a default value for the unchecked status used on output display of the checkbox value.

Other attributes may be inherited from the parent property.

Usage

Standalone

  • In your template forms: you can use the data-input tag to capture data for this property. 
        <xar:data-input type="checkbox" hasvalue="I agree" name="myfield"
            [attr1="value1",...] />

Defined Data Property

  1. Defining your data property : In the Configuration screen for this property you can add values for 'Checked box value' and 'Unchecked box value'.
  2. Using the data-input tag: you can use the data-input tag to capture data for this property and pass in attributes for the data property input. 
        <xar:data-input property="$properties['checkbox']"
            hasnovalue="Not accepted" [attr1="value1",...] />

Example

        <xar:data-input type="checkbox" id="agreetoterms"
            name="acceptterms"    />
