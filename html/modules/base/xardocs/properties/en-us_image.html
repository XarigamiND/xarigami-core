Description

The Image property provides input options for an image. Input can be either an image URI in a textbox, selection of an image from from a local image repository, or upload of an image. The method depends on the configuration.

On output display, the image is displayed.

The base file name of the image is saved to the database without path information.

Attributes

  • image_source: [default='url'][url|local|upload]
    Sets the input source of the image as either an image URI, browsing to a local image file or an image file upload.
  • extensions: [default='gif,jpg,jpeg,png,bmp']
    A listing of allowable file type extension for the image.
  • maxsize: [default='']
    The maximum file size allowed for upload.
  • basedir: [default='']
    The local directory to browse from (local), or directory to upload the image to for File upload source type.
  • longname: [default=false]
    Option to display the file extension in the selector box (local) and on display.
  • display: [default=0]
    Display an image file on input.

Other attributes may be inherited from the Textbox parent property or added for upload (file upload) or local (filelist) image source.

Usage

Standalone

  • In your template forms: you can use the data-input tag to capture data for this property. 
        <xar:data-input type="image" basedir="var/uploads" name="myimage" image_source="local"
            [attr1="value1",...] />

Defined Data Property

In the Dynamic Data Configuration:

  • you can add a default value in the 'Default' input box. Enter other configuration values as required such as a minimum (min) or maximum (max) value for this property.

Using the data-input tag: you can use the data-input tag to capture data for this property and pass in attributes for the data property input. 

        <xar:data-input property="$properties['image']"
             [attr1="value1",...] />

Example

        <xar:data-input type="image" id="myavatar"
           basdir="var/uploads" file_ext='png,gif,jpg'  />
