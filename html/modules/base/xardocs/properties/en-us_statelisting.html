Description

The StateListing property provides a drop down selection list of states or provinces for one or more specified countries.

The State or Province names are displayed in the form in a dropdown while the ISO-3166-2 State or Province code is the value captured in forms and saved in the database.

On output the State or Province name is displayed.

Attributes

  • country: [default = 'us,ca,au' to support legacy installs]
    The two character country code abbreviation as defined in ISO 3166. You may specify a single country code or a list separated by commas. Unsupported country codes are ignored.
  • firstline: [default = 'Please select']
    This attribute is inherited from the parent 'dropdown' selection property, and enables you to specify a first line in the selection list displayed as an instruction to the user.

Other attributes may be inherited from the Dropdown List parent property.

Usage

Standalone

You can use the statelisting type in forms without the need to define a property to capture data. 

        <xar:data-input type="statelisting" country="$country" name="myfield"
            [attr1="value1",...] />

Defined Data Property

  • In the Dynamic Data Configuration: you can add country options when defining your data property by adding the country code(s) to the Other field in the validation page.
  • In your template forms: you can use the data-input tag to capture data for this property. 
        <xar:data-input property="$properties['statelisting']"
            country="$country" [attr1="value1",...] />

Example

        <xar:data-input type="statelisting" id="countrystate"
            name="countrystate" country="au,sg"  firstline="Please select your State" />
