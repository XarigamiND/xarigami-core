Description

The floatbox (Number Box - float) property provides an input box for entry of float type numeric input. This means you can enter positive or negative numbers, with or without decimal points.

The output display for integerbox displays the input value plus any other display configurations you have set. This can include a prefix or suffix for the number, decimal separator, group separator or number of decimal points.

The value saved in the database is the value as entered, without any other display characters. If precision is defined, the number saved will be that number value after precision is applied with a decimal point (.) replacing any other defined decimal separator.

Attributes

  • min: [default = '']
    This is the minimum value of the number required for input.
  • max: [default = '']
    This is the maximum value of the number required for input.
  • default: [default = '']
    This is default number at item creation.
  • precision: [default = '']
    This is number precision here used as number of significant digits after the decimal point. Numbers with more than the specific precision will be rounded down.
  • prefix: [default = '']
    This prefix will be displayed before the value on output display.
  • suffix: [default = '']
    This suffix will be displayed after the value on output display.
  • decimal_sep: [default = '']
    This is the Decimal separator which will be used for the number. Note that if you use a decimal separator, all numbers entered by the user will be forced to the number of decimal points given by the prevision value, and either rounded or padded with zeros as necessary. If you do not supply a decimal separator, the user can enter numbers with and without numbers after the decimal point and the numbers will be displayed and saved as such

Other attributes may be inherited from the parent Textbox property as well as common attributes such as class, placeholder and so on.

Usage

Standalone

You can use the integerbox type in forms without the need to define a property to capture data. 

        <xar:data-input type="integerbox" default="$defaultnumber" name="myfield"
            [attr1="value1",...] />

Defined Data Property

In the Dynamic Data Configuration:

  • you can add a default value in the 'Default' input box. Enter other configuration values as required such as a minimum (min) or maximum (max) value for this property.
  • if you check the 'Allow empty' checkbox, this will permit users to skip entry of data in this input box. If the 'Allow empty' checkbox is unchecked, users will have to enter a value. Zero (0) is considered a valid integer and not empty.

In your template forms: you can use the data-input tag to capture data for this property. 

        <xar:data-input property="$properties['integerbox']"
            [attr1="value1",...] />

Example

        <xar:data-input type="integerbox" id="myfield" name="myfield"
        min="12" max="99" default="18" />
