Description

The Radio property provides a set of radio buttons with values that are user definable.

The value stored in the database depends on whether you have defined an associative or non-associative array of values for your radiobuttons. The id not the value of an option is used. eg In the case of options defined as 1,Yes;2,No;3,Maybe, the 1, 2 or 3 is stored in the database. If you define your options as Yes;No;Maybe then those values become the ids themselves and are stored in the database on selection.

The display value of the chosen option is always displayed on output, not the id.

Attributes

  • class: [default='']
    The class can be specified and will be applied to the select tag that is generated from the dropdown data property.
  • function: [default='']
    Add a xarigami function that delivers an array of values to populate the radio button options.
  • options: [default=array()]
    This list can be specified as an array of values to populate the radio buttons eg 
     1,Yes;2,No;3,Maybe
    or 
     Yes;No;Maybe

Other attributes may be inherited from the Drop Down Select parent property. Not all of these may be applicable to Radio Buttons.

Usage

Standalone

You can use the radio type to capture data in forms without the need to define a property. 

        <xar:data-input type="radio" name="myfield" id="myfield" options="$myoptions"
            [attr1="value1",...] />

The 'options' array is in the form of a non-associative or associative array of id=>'value' options eg:

array(1=>'Yes', 2=>'No', 3=>'Maybe')

Note that 'validation' is an equivalent legacy attribute and still honoured at this time.

Defined Data Property

You can capture data for this bound property in your templates with the data input tag. 

        <xar:data-input property="$properties['myradiobuttons']"
            [attr1="value1",...] />

Example

        <xar:data-input type="radio" id="myresponses"
            name="myresponses />
