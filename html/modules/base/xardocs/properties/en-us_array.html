Description

The Array property (type 999) provides a GUI for input of an associative or non-associative array. This can be defined by passing in attribute values in the data input tag or in the configuration of a bound property.

The array is displayed by default as a list of option-value pairs, one per line with option value pairs separated by a comma.

The database value is stored as a serialized array of option value pairs.

Attributes

  • associative_array: [default=0]
    Defines whether an associative array is used or not. A value of 1 will provide an key-value input for the array data property, and a minimum of two columns. A non-associative array will display one column by default.
  • allowinput: [default=0]
    Defines whether the user can add (1) or delete (2) rows from the array. The default is 0 so only the original number of defined rows are displayed.
  • rows: [default=1]
    The number of rows to display at start.
  • maxrows: [default=10]
    The maximum number of rows allowed. A value of 0 enables unlimited rows.
  • columns: [default=1]
    The number of columns to display not including the key.
  • keylabel: [default=""]
    The label for the Key column
  • valuelabel: [default=""]
    The label for the Value column. If there is more than one column, multiple value labels can be entered separated by semicolons (;)
  • fixedkeys: [default=0]
    If fixed keys is set to 1, users cannot edit keys once the value is input and saved.

Other attributes may be inherited from the parent property.

Usage

Standalone

You can use the array type to capture data in forms without the need to define a property. 

        <xar:data-input type="array" name="myarray" id="myarray" associative_array="1"
           rows="2" columns="2" allowinput="2" keylabel="Option" valuelabel="Value" [attr1="value1",...] />

Defined Data Property

You can capture data for this bound property in your templates with the data input tag. 

        <xar:data-input property="$properties['arraylist']"
            [attr1="value1",...] />

Example

        <xar:data-input type="array" id="myitems"
            name="myitems" rows="1" columns="2" associative_array="1" keylabel="Pet name" valuelabel="Animal type" />
