Description

The Textbox data property provides an input box for text entry.

The property display output proves the text value prepared for allowed HTML output

The actual value of the input text is saved in the database,

Attributes

  • default: [default = '']
    This is the default text displayed in the input box.
  • min: [default = '']
    This is the minimum number of characters required for valid input.
  • max: [default = '250']
    This is the maximum number of characters accepted for valid input.
  • regex: [default = '']
    This is a regular expression that is applied to the text and used to validate input.
  • size: [default = 50]
    This is the character width of the input box on display.
  • autocomplete: [default = 'on']
    Turn of autocomplete of this field if available. This attribute is valid only HTML5 only.
  • placeholder: [default = '']
    This is placeholder text displayed in the body of the textbox on input which usually would be used to give the user a clue to input. This attribute is valid only HTML5 only.
  • class: [default = 'xar-form-text*']
    If defined, the class will be added to the text input tag and applied otherwise a default class is used determined by the size of the text box. The fallback default can be overridden by adding an empty space character to the class input box.

Other attributes may be inherited from the parent property.

Usage

Standalone

You can use the textbox type in forms without the need to define a property to capture data. As well as the attributes available in GUI configurations above, standalone properties also accept valid HTML5 and ARIA WAI attributes passed in the tag. 

        <xar:data-input type="textbox" default="$defaulttext" name="myfield"
            [attr1="value1",...] />

Defined Data Property

In the Dynamic Data Configuration:

  • you can force a value to be unique by checking the Unique checkbox. This should be used with caution. You should evaluate whether this is suitable when working with large amounts of data and be especially careful using on long or complex field values It will affect performance.
  • you can set other validation and display rules in the configuration such as placeholder text, regex requirements and so on.

When you define this property you can add a default text value in the 'Default' input box.

In your template forms you can use the data-input tag to capture data for this property. 

        <xar:data-input property="$properties['textbox']"
            [attr1="value1",...] />

Example

        <xar:data-input type="textbox" id="myfield" name="myfield"
        min="4" max="10" default="Any text" />
