Description

The integerbox (Number Box) property provides an input box for entry of whole (integer) numbers.

The output display for integerbox displays the input value

The value as input is saved in the database.

Attributes

  • min: [default = '']
    This is the minimum value of the number required for input.
  • max: [default = '']
    This is the maximum value of the number required for input.
  • default: [default = '']
    This is default number at item creation.

Other attributes may be inherited from the parent Textbox property as well as common attributes such as class, placeholder and so on.

Usage

Standalone

You can use the integerbox type in forms without the need to define a property to capture data. 

        <xar:data-input type="integerbox" default="$defaultnumber" name="myfield"
            [attr1="value1",...] />

Defined Data Property

In the Dynamic Data Configuration:

  • you can add a default value in the 'Default' input box. Click on the Validation entry to add a minimum (min) or maximum (max) value for this property.
  • if you check the 'Allow empty' checkbox, this will permit users to skip entry of data in this input box. If the 'Allow empty' checkbox is unchecked, users will have to enter a value. Zero (0) is considered a valid integer and not empty.

In your template forms: you can use the data-input tag to capture data for this property. 

        <xar:data-input property="$properties['integerbox']"
            [attr1="value1",...] />

Example

        <xar:data-input type="integerbox" id="myfield" name="myfield"
        min="12" max="99" default="18" />
